LES RESEAUX INFORMATIQUES

1-Reseaux network:

Un réseau est constitué de deux ordinateurs ou plus, connectés selon des règles spécifiques afin de partager des ressources quelles que soient les longues distances qui les séparent, car ces réseaux sont apparus et se sont développés au fil du temps en raison de la nécessité de connecter des ordinateurs et des stations entre eux, et puis plusieurs autres fonctionnalités développées en accord avec la technologie moderne, et Le réseau est basé à la fois sur l'interconnexion et la connectivité.


2-Caracteristique de base:

Les propriétés de base sont : la topologie, le débit exprimé en bits, la distance maximale et le nombre maximal de nœuds interconnectés.

3-Terminologie reseaux:

Les termes de réseau incluent les postes de travail, les nœuds, les paquets et les réseaux hétérogènes.

4-Elements d'un reseau:

Ses éléments sont les ordinateurs, les logiciels, les supports du LAN et WAN, et les équipements d’interconnexion du LAN.

5-Classification des reseaux:

- Ouverture, où l'on retrouve le réseau externe, les réseaux publics, et l'Internet interne privé.
- Les supports de connexion, où l'on distingue les réseaux filaires utilisant des câbles, tandis que les réseaux sans fil sont utilisés par diverses ondes et Bluetooth.




- Le type d'organisation où l'on retrouve les réseaux peer-to-peer et les réseaux client/serveur.
- En fonction de la distance ou de la portée des réseaux WAN LAN MAN


6-Le type de topologie:

En ce qui concerne le type de topologie, il existe des réseaux en bus, en étoile, en anneau et maillée (à emboîtement).

Le Modèle OSI

1)Les Composants matériel d’un Réseau informatique:

1-Les cartes réseau:

Constitue l’interface physique entre l’ordinateur et le support de communication.


2-SWITCH:



Partage les données uniquement avec le destinataire de ces données.

3-HUB:

Equipement physique a plusieurs ports servant a relier plusieurs ordinateurs.Il Dirige les données et les partagés avec l’ensemble des machines connecté a lui

2)Le Modèle OSI:OSI (Open Systems Interconnection),Les sept couches du modèle OSI sont les suivantes

  • La couche physique;
  • La couche liaison de données;
  • La couche réseau ;
  • La couche transport;
  • La couche session;
  • La couche présentation;
  • La couche application;

3) Le Modèle TCP/IP

Le modèle TCP/IP (Transmission Control Protocol/Internet Protocol) est un modèle de référence pour les communications de réseau, qui est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes. il est dévisé en quatre couches principales :


  • La couche liaison de données
  • La couche réseau
  • La couche transport
  • La couche application

4) l'adressage:

Les communications dans un réseau sont régies par des règles, notamment l'adressage IP, l'architecture et les protocoles. L'adressage IP est un numéro unique qui identifie chaque ordinateur dans un réseau. Il existe deux modes d'adressage IP : IPv4 avec des adresses de 32 bits et IPv6 avec des adresses de 128 bits représentées en hexadécimal.


5) L’adresse IPV4 :

Chaque adresse IPv4 est divisée en deux parties :


  • la partie pour le réseau
  • la partie hôtes
Les différents types d’adresse IPV4:


  • L’adresse réseau
  • L’adresse hôtes
  • Le masque de réseau et de sous réseaux

Système de Gestion de Version
Qu'est-ce que Git?

Git est un logiciel de gestion de version créé en 2005 par Linus Torvalds, le créateur de Linux.Il permet de conserver un historique des modifications effectuées sur un projet et de revenir à une ancienne version en cas de problème. Les logiciels de gestion de versions sont essentiels pour la gestion de projets et travailler en équipe de manière efficace. Git est le leader incontesté parmi les logiciels de gestion de versions et est indispensable pour tout développeur.

A quoi sert concrètement un système de gestion de version ?

Un système de gestion de version permet de conserver un historique des modifications effectuées sur un projet et de coordonner le travail entre plusieurs personnes. Dans un contexte de travail collaboratif sur un site web, cela permet de s'assurer que les fonctionnalités développées par différents développeurs ne rentrent pas en conflit et que chaque développeur possède une version actualisée du site pour travailler. Les logiciels de gestion de version sont basés sur deux modèles : le modèle centralisé, où la source du code du projet est hébergée sur un serveur distant central, et le modèle décentralisé, où chaque utilisateur télécharge et héberge l'intégralité du code source du projet sur sa propre machine. Le modèle décentralisé est populaire grâce à sa simplicité, sa flexibilité et sa sécurité accrue.

Les deux modèles des logiciels de gestion de version : modèle centralisé vs modèle décentralisé

Les logiciels de gestion de version sont aujourd’hui tous construits sur l’un des deux modèles suivants le modèle centralisé et le modèle distribué. Dans le modèle centralisé, le code source est hébergé sur un serveur distant central, et les utilisateurs doivent se connecter à ce serveur pour travailler sur le code. En revanche, dans le modèle distribué, chaque utilisateur doit télécharger et héberger l'intégralité du code source du projet sur sa propre machine. Le texte explique que le modèle distribué, popularisé par Git, présente plusieurs avantages notables , notamment en termes de simplicité et de flexibilité du travail, car chaque utilisateur peut travailler en ligne sur sa propre machine sans être connecté en permanence à un serveur central. De plus, le modèle distribué offre un niveau de sécurité élevé, car chaque utilisateur possède une copie complète du code du projet, pouvant être utilisée comme backup en cas de corruption du serveur central.

Qu’est-ce que GitHub ?

un logiciel de gestion de version qui permet d'enregistrer les différentes modifications effectuées sur un projet et de revenir à une version précédente. Il explique que le dépôt, c'est la copie de l'intégralité des fichiers d'un projet et de leur version située sur le serveur central, et que GitHub est le plus grand hébergeur de dépôts Git au monde, offrant la possibilité d'héberger des dépôts Git en ligne et de proposer de nouvelles fonctionnalités à des dépôts publics. En somme, Git est un logiciel de gestion de version et GitHub est un service en ligne d'hébergement de dépôts Git qui sert de serveur central pour ces dépôts.

Installation de Git

La façon la plus simple d’installer Git est de télécharger la dernière version sur le site officiel http://git-scm.com/downloads , d’ouvrir le fichier téléchargé et de suivre les instructions à l’écran en laissant toutes les valeurs par défaut.

Paramétrage de Git

Le paramétrage de Git est important pour enregistrer certaines données, telles que le nom d'utilisateur et l'adresse mail pour ne pas les fournir à nouveau plus tard. La commande Git config permet de voir et modifier les variables de configuration de Git, y compris le nom d'utilisateur et l'adresse mail. L'option --global est utilisée pour indiquer à Git que ces informations doivent être utilisées globalement pour tout projet Git. Les commandes git config --global user.name et git config --global user.email sont utilisées pour renseigner le nom et l'adresse mail de l'utilisateur respectivement, et la commande git config user.name et git config user.email est utilisée pour vérifier si les informations ont été enregistrées.

Démarrer un dépôt Git

un dépôt Git est une copie des fichiers d'un projet importée dans Git, et qu'il existe deux façons de créer un dépôt : en important un répertoire existant dans Git ou en clonant un dépôt Git existant. Avant de voir comment faire cela, il est recommandé de comprendre la gestion des informations dans Git et son fonctionnement général.

La gestion des informations selon Git

Git gère les données sous forme d'instantanés ou "snapshots". Lorsqu'un état de projet est enregistré dans Git, il prend un instantané du contenu de l'espace de travail à ce moment-là et enregistre une référence à cet instantané. Chaque instantané est stocké localement dans une base de données. Cela permet à la plupart des opérations de Git d'être effectuées localement, sans avoir besoin d'être connecté à un serveur central distant, ce qui rend le travail plus rapide et agréable.

Les états des fichiers

Git suit les modifications sur les fichiers d'un projet en les classant comme "suivis" ou "non suivis". Lorsqu'un fichier est suivi, il appartient au dernier instantané capturé par Git et est enregistré en base. Lorsqu'un dépôt est créé, tous les fichiers sont non suivis et doivent être indexés et validés. Un fichier suivi peut avoir l'un des trois états : "modifié", "indexé" ou "validé". Les fichiers modifiés sont considérés comme "modifiés" jusqu'à ce qu'ils soient indexés, tandis que les nouveaux fichiers sont considérés comme "non suivis". Indexer un fichier signifie l'ajouter au prochain instantané, tandis que valider un instantané signifie stocker son état actuel en base. Le cycle peut recommencer en demandant à Git de prendre un nouvel instantané.

Les zones de travail

Tout projet Git est composé de trois sections : le répertoire de travail, la zone d'index et le répertoire Git. Le répertoire de travail correspond à une extraction unique d'une version du projet. La zone d'index stocke les informations sur ce qui fera partie du prochain instantané ou commit, tandis que le répertoire Git est l'endroit où Git stocke les métadonnées et la base de données des objets du projet. Le processus de travail consiste à travailler sur les fichiers dans le répertoire de travail, les indexer, puis les valider pour les ajouter au répertoire Git.

Créer un dépôt Git à partir d’un répertoire existant

comment créer un dépôt Git à partir d'un répertoire existant contenant des fichiers en local. Tout d'abord, il est recommandé d'utiliser la commande "git init" pour initialiser un dépôt Git. Ensuite, la commande "git status" permet de vérifier l'état des fichiers du répertoire et de déterminer quels fichiers ne sont pas encore sous suivi de version ("untracked"). Pour ajouter des fichiers à l'index et les indexer, on utilise la commande "git add", qui peut indexer des fichiers déjà sous suivi de version et placer sous suivi des fichiers non suivis. Enfin, la commande "git commit" permet de valider les fichiers et de les ajouter à la base de données en fournissant un message descriptif.

Cloner un dépôt Git

comment cloner un dépôt Git existant en utilisant la commande git clone. Pour ce faire, il faut créer un nouveau dépôt sur GitHub et exécuter les commandes git branch -M main, git remote add origin https://github.com/BouchekirRedouane/ExmpleCloneGit.git et git push -u origin main pour envoyer les fichiers au dépôt distant. En vérifiant le dépôt sur GitHub, on peut confirmer que les fichiers ont été transférés avec succès

Ajouter ou modifier des fichiers dans un projet Et actualiser notre dépôt Git:

Dans un projet Git, nous utilisons les commandes "git add" et "git commit" pour enregistrer les modifications de fichiers dans l'historique des versions. Le "git commit" se base sur l'état des fichiers au moment du dernier "git add". Pour enregistrer la dernière version d'un fichier, il est important d'effectuer un "git add" juste avant le "git commit". Une option pratique est d'utiliser "git commit -a" pour ajouter et valider automatiquement les fichiers modifiés déjà suivis.

consulter l'historique des modifications Git

La commande "git log" permet de consulter facilement l'historique des modifications dans Git. Elle affiche la liste des commits effectués, du plus récent au plus ancien, avec des informations telles que la somme de contrôle SHA-1, le nom de l'auteur, l'e-mail, la date et le message du commit

annuler des modifications apportées à un fichier

Un avantage clé de l'utilisation d'un logiciel de gestion de version est la capacité à effectuer un "rollback", c'est-à-dire à revenir à un état précédemment enregistré d'un projet. Après un commit, si des modifications ne fonctionnent pas comme prévu, on peut utiliser la commande git checkout -- nom-du-fichier ou git restore pour revenir à l'état précédent du fichier.

Qu'est ce qu'une branche ?

Git facilite la création de branches pour développer de nouvelles fonctionnalités sans affecter le projet principal. Contrairement à d'autres systèmes, Git utilise des pointeurs vers des commits pour représenter les branches. Chaque commit contient des informations sur l'auteur, le message et un pointeur vers le contenu indexé, ainsi que des références aux commits précédents. La branche par défaut, appelée "master", se déplace automatiquement à chaque nouveau commit. En résumé, créer une nouvelle branche dans Git signifie simplement créer un nouveau pointeur sans copier tout le répertoire de travail.
